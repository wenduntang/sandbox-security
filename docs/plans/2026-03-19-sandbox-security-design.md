# 沙箱安全运行环境 - 设计文档

**日期**: 2026-03-19  
**状态**: 已批准

---

## 1. 目标

构建一套沙箱安全运行环境，用于**隔离运行不可信 Web 服务或 API**，支持：
- Docker 与进程级双重隔离（可组合/备选）
- 三种正逆向验证：环境/配置、白名单+审计、资源配额
- 三种使用方式：CLI、Python 库、Web 界面

---

## 2. 架构

```
┌─────────────────────────────────────────────────────────┐
│                    Web UI (FastAPI + 前端)               │
├─────────────────────────────────────────────────────────┤
│                    CLI (click/typer)                     │
├─────────────────────────────────────────────────────────┤
│               sandbox-core (Python 库)                    │
│  ┌─────────────┬─────────────┬─────────────────────────┐ │
│  │ Runtime     │ Verifier   │ Audit / Resource        │ │
│  │ (Docker/    │ (正向+逆向) │ Tracker                 │ │
│  │  Process)   │             │                         │ │
│  └─────────────┴─────────────┴─────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

---

## 3. 三种验证流程

| 类型 | 正向（启动前） | 逆向（结束后） |
|------|----------------|----------------|
| **环境/配置** | 检查隔离环境、挂载、网络 | 对比启动前后文件/配置，检测篡改 |
| **白名单/审计** | 加载允许的操作白名单 | 审计日志回放，校验是否越权 |
| **资源** | 设置 CPU/内存/网络配额 | 对比实际用量与配额，超限告警 |

---

## 4. 运行时策略

- **优先**：Docker 容器（隔离强）
- **备选**：进程级隔离（`subprocess` + `resource` / `cgroups`）
- 自动检测 Docker 可用性，不可用时回退到进程模式

---

## 6. 部署位置

- 项目根目录：`~/apps/sandbox-security/`
- 虚拟环境：`~/.sandbox-security-venv/`

---

## 7. 错误处理

- 正向验证失败：拒绝启动，返回明确错误码
- 逆向验证失败：标记为不安全，记录告警
- 资源超限：终止进程，记录违规

---

## 8. 测试策略

- 单元测试：各验证器、运行时
- 集成测试：完整启动→运行→结束→验证流程
- 安全测试：尝试越权、篡改、资源耗尽，验证防护有效
